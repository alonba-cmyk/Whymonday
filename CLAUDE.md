# Working preferences

- Always commit and push new/updated files to the git repo (not just deliver them as claude.ai artifacts or local files). The user wants everything version-controlled, including exploratory/review files (e.g. slide-variant comparison pages), not only final deck files.
- After pushing a branch, open a PR and merge it straight to main without waiting for confirmation, so GitHub Pages redeploys and the change goes live. No need to ask each time.
- When giving an opinion, recommendation, or ranking between options (copy, design, naming, etc.), do real analysis first — check how the option connects to other specific content already in the deck/codebase, not just a surface-level gut reaction based on brevity or tone. Don't undersell an option in a first pass only to find its real justification later when pushed on it.
